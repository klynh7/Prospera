Transaction Change Summary
==========================

Backend files changed:
- backend/models/transaction.js
  - Added `transaction_type` enum field to Transaction model.

- backend/models/transaction_detail.js
  - Added `transaction_type` enum field to TransactionDetail model.

- backend/controllers/transactionController.js
  - Updated createTransaction to accept per-item `transaction_type`.
  - Validates item-level transaction type (`buy` or `sell`).
  - Applies stock changes per item:
    - decrement stock for `sell`
    - increment stock for `buy`
  - Saves transaction details with item `transaction_type`.
  - Includes transaction details in history response.

Frontend files changed:
- frontend/src/pages/Transaction.jsx
  - Added item-level `transactionType` to cart items.
  - Auto-fills `modal` and `harga jual/harga beli` when a product is selected.
  - Sends `transaction_type` for each item in the checkout payload.
  - Added history detail modal popup to inspect past transactions.
  - Displays mixed transaction type history as `MIXED` when a transaction contains both buy and sell items.

Migration requirement:
- backend/migrations/20260530153000-add-transaction-type-to-transactions.js
  - Adds `transaction_type` column to `Transactions` table.

- backend/migrations/20260530172000-add-transaction-type-to-transaction-details.js
  - Adds `transaction_type` column to `Transaction_details` table.

Exact behavior for buy/sell mixed transactions:
- The cart can contain both `buy` and `sell` items in a single checkout.
- Each item stores its own `transaction_type`.
- The backend updates stock per item, not per transaction header.
  - `sell` items decrease stock.
  - `buy` items increase stock.
- The transaction header still stores one `transaction_type` value, but the history UI shows `MIXED` when detail items contain both types.
- Transaction history includes detail rows so users can review each product and its buy/sell type in a popup.
